# Memory Index

This is the router for project memory. At the start of a session, skim this index.
Read a topic file only when it clearly applies to the current request. Keep the check
lightweight — don't read everything every time.

When something durable is learned (a preference, a setup fact, a platform gotcha, a
project decision), add or update the relevant topic file and list it here.

| Topic file | Read it when... | Last updated |
|---|---|---|
| `preferences.md` | Any session — communication style, output format, defaults | 2026-06-12 |
| `setup.md` | Technical work — environment, tools, prior builds, the business (True North AI) | 2026-06-12 |
| `platform-gotchas.md` | Working in Claude.ai Projects or Claude Code; where things live; odd behavior | 2026-06-12 |
| `project-email-triage.md` | Any work on the email overload / assisted-triage project | 2026-06-12 |
| `deployment-intake.md` | Onboarding a new executive / standing up a deployment | 2026-06-12 |

## Conventions
- One topic per file. Keep entries short and durable (facts that stay true across sessions).
- Don't store secrets, credentials, tokens, or sensitive personal data here.
- Date each entry or file so staleness is visible.
